Solr学习（二）

发表于 2019-08-30 更新于 2019-09-03 分类于 Solr 阅读次数：
本文字数： 1.8k 阅读时长 ≈ 2 min

记录学习Solr过程中所遇到的问题及解决方法

问题

solr无法添加core

错误显示

Linux安装好solr后，新建core时报错，提示缺少配置文件：Error CREATEing SolrCore ‘collection1’: Unable to create core [collection1] Caused by: Can’t find resource ‘solrconfig.xml’ in classpath or ‘/usr/solr/solrhome/new_core’

解决方法

将上图的文件复制到/usr/solr/solrhome/new_core中，就是所要创建的core目录中（new_core为存放core的文件夹名，可根据需要自行修改）

cp -r /usr/solr/solr-7.7.2/server/solr/configsets/_default/conf /usr/solr/solrhome/new_core

接着重启Tomcat服务器

再去页面创建一个与上面新建文件夹名一致的core即可，可见core添加成功！

添加成功后，会在该core的目录下生成data文件夹与core.properties文件。

solr-4.10.3安装中文分词器出错

错误原因：使用solr-4.10.3过程中，安装IK中文分词器后出现core不存在的问题。

解决方法：该版本兼容 IKAnalyzer2012_u6.jar会出现问题，换用IKAnalyzer2012FF_u1.jar即可解决问题。

solr-7.7.2使用IK中文分词器出错

在Linux环境下，安装好IK中文分词器后，在使用分词时会出现java.lang.AbstractMethodError错误。

错误原因：当前Solr版本7.7.2，IK-analyzer版本过低，现在在网上下的IK分词器版本基本都是IKAnalyzer2012FF_u1，需要一个高版本的适配。

解决方法：现更换ik-analyzer-solr5-5.x版本后，分词器正常使用（也可更换5及5以上的版本）。

solr-7.7.2导入数据库数据显示不全

solr版本：7.7.2

user表结构

根目录下managed-schema配置（id已有，无需添加）

使用dataimport插件导入数据库数据后，查询发现只显示id，其余新建的域均无显示

网上查找了很多的资料，都是版本不一、解决方法无效等等问题，一直都没有解决，后来想到新建的域在Analysis页面字段类型应可以被查询到，但字段还是无法被查询出来

（Solr 5.3可能会出现相同的问题，原因是默认自带的managed-schema和schema.xml冲突所致，只需保存前者）

通过后台管理页面来添加新的域，进入Analysis页面，点击Schema Browser

进入到域添加页面，点击Add Field，根据需要填写配置点击Add Field即可。

在core的根目录，vim conf/managed-schema可查看到新增的域信息

重启tomcat，重新导入数据，字段已能正常显示

连接SolrJ失败

打开Solr后台管理页面链接为：http://ip地址:8080/solr/index.html#/，连接SolrJ过程中，使用了相同的链接，然后打开页面出现500错误。

解决方法：更换Solr地址–>http://ip地址:8080/solr/core名称，更改后即可成功连接SolrJ，如该问题还未解决，可查看SolrJ相关的jar包是否导入成功。

以上均是在学习Solr过程中遇到的问题，问题并不可怕，不要气馁，只需要拥有一颗坚持的心。
